# Neo Brutal UI 全面翻新設計規格

日期：2026-07-22

## 1. 目標

將現有加班薪資計算器全面翻新為高可讀性的明亮 Neo Brutal 介面，同時保留既有計算、資料、PWA 與互動行為。

核准方向：

- 明亮米白底 Neo Brutal。
- 單頁垂直工作流。
- 繁體中文字必須清楚可見。
- 使用 Uiverse Galaxy 作為元件視覺來源，但不得原封不動匯入整套元件。
- 不改寫薪資計算或資料架構。

## 2. 現況與限制

專案是靜態單頁 PWA。`index.html` 同時包含主要 DOM、商業資料、計算函式與狀態操作；`styles.css` 提供目前深色控制台外觀；`motion.js` 提供 `window.UiMotion` 動畫 API。

工作樹已有未提交的 `index.html`、`styles.css`、`service-worker.js` 與 `uiverse.css` 變更。實作必須保留這些修改，禁止以基準檔或歷史提交覆蓋。

翻新不得改變：

- 既有 DOM IDs。
- LocalStorage keys。
- 全域函式與 inline handler 對應。
- 加班倍率、保險級距、假日、颱風假與補班資料。
- 日期區間、加班記錄、匯入、獎金與薪資計算流程。
- `window.UiMotion` 公開 API。
- PWA 離線能力。

## 3. 資訊架構

維持單頁垂直資料流：

1. 本期狀態摘要。
2. 選擇薪資週期。
3. 填寫加班月曆。
4. 填寫薪資、請假與扣款資料。
5. 查看薪資計算結果。

右側加班記錄抽屜與阻斷性 modal 保持獨立浮層，不加入新的頁面或步驟狀態。

## 4. 視覺系統

### 4.1 色彩

- 頁面底色：溫暖米白。
- 主要文字：近黑色。
- 邊框：黑色粗框。
- 主操作與結構重點：藍色。
- 成功與已填寫：綠色。
- 警告與實領：黃色。
- 錯誤與刪除：紅色。
- 資訊分類可使用粉色或青綠色，但必須維持黑字可讀性。

顏色不可作為唯一狀態訊號；狀態必須同時包含文字或圖示。

### 4.2 字體與可讀性

使用本機繁中字體堆疊：`Microsoft JhengHei`、`PingFang TC`、系統 sans-serif。禁止載入外部字型。

- 正文至少 14px。
- 主要輸入至少 16px，避免手機瀏覽器自動縮放。
- 小型輔助標籤至少 12px。
- 主要標題使用 700–900 字重。
- 正文不得使用低對比淺灰。
- 不在文字本體套用 glitch、模糊、掃描線或多層陰影。
- 裝飾陰影只作用於容器與控制項外框。

一般正文與背景至少達 WCAG AA 對比；焦點、錯誤與重要結果需有清楚的形狀或雙層外框。

### 4.3 形狀與動態

- 卡片使用 3–4px 黑色邊框與偏移硬陰影。
- 圓角保持小幅度或直角，不使用大面積柔和玻璃效果。
- 按鈕按下時向右下移動，陰影同步縮短，模擬實體按鍵。
- 輸入 focus 只改變邊框與外框，不移動文字或造成抖動。
- GSAP 僅負責區塊進場、抽屜與 modal。
- `prefers-reduced-motion` 下移除位移、掃光與彈性動畫。

## 5. Uiverse Galaxy 元件配對

Uiverse 原始元件只作為視覺與互動參考。實作時必須抽取必要部分、重新命名選擇器、套用專案 design tokens、補齊無障礙，並保留來源註記。

| 現有介面 | 目標樣式 | Galaxy 來源方向 |
|---|---|---|
| 頁首與狀態摘要 | 粗框彩色狀態磚 | Brutalist card pattern |
| 日期與薪資輸入 | 高對比硬陰影輸入框 | `Inputs/0xnihilism_calm-baboon-55.html`，移除 glitch |
| 主要按鈕 | 位移、硬陰影、有限掃光 | `Buttons/0xnihilism_fast-cat-82.html` |
| 一般內容卡片 | 粗框、分隔線、偏移陰影 | `Cards/0xnihilism_thin-crab-36.html` |
| 快捷時數鍵 | 實體鍵帽按壓效果 | 由 Brutalist button pattern 縮小適配 |
| 請假選項 | 高對比 checkbox/radio，全列可點 | Galaxy controls 經可存取性改寫 |
| 成功／警告／錯誤 | 實色通知卡，含圖示與文字 | Galaxy notification pattern |
| Tooltip | 黑底白字或黃底黑字說明 | Galaxy tooltip pattern，手機改用可點擊說明 |
| Loading | 只用於真實等待狀態 | `loaders/0xnihilism_brown-puma-30.html` 經降動態適配 |

禁止使用 Uiverse 原始的通用 `.card`、`.button`、`.input` 類名。所有抽取樣式必須加上專案命名空間或映射到既有明確類別。

## 6. 各介面設計

### 6.1 頁首與摘要

頁首顯示品牌、用途與版本。摘要區顯示結算區間、月曆加班時數、記錄筆數與匯入狀態。每個摘要磚使用不同實色背景，但共用相同邊框、字級與資料格式。

### 6.2 日期區間

桌機維持雙欄日期輸入；手機改為單欄。主要操作按鈕占據足夠寬度，按下後沿用既有 `generateCalendar()` 流程。

### 6.3 加班月曆

桌機維持七欄 table DOM。日期格改為小型 Brutal 卡片：

- 一般工作日：白底。
- 週末與假日：不同實色或醒目邊線。
- 已填寫：綠色狀態。
- 異常或超出合理範圍：紅色警告。

日期格保留日期、類型、時數輸入與 2／4／8／清除快捷鍵。手機沿用既有「table DOM 轉每週垂直清單」方式，不新增第二份月曆 DOM。

### 6.4 薪資與請假表單

桌機維持收入資料與扣款／請假資料雙欄；手機全部單欄。標籤固定在輸入框上方。唯讀欄位必須顯示「固定金額」標記，不能只靠灰色外觀表達。

請假類型使用全列可點擊控制項，清楚顯示全薪、半薪或其他計算說明。

### 6.5 薪資結果

結果區先顯示大型黃色實領金額，再呈現摘要磚與收入／扣款明細。桌機為雙欄帳目；手機依序顯示實領、收入、扣款。

所有金額使用等寬數字特性與一致格式。收入、扣款與總計除了顏色外，需有標題與正負語意。

### 6.6 加班記錄抽屜

桌機維持右側抽屜；手機改為全螢幕工作板。標題與關閉鍵固定在頂部。新增、篩選、獎金進度、匯入與記錄列表各自使用有明確標題的小卡。

保留新增、編輯、刪除、匯入、清除與篩選行為。刪除按鈕使用紅色，但仍保留文字或可存取名稱。

### 6.7 Modal 與通知

欄位錯誤盡量顯示在欄位附近。modal 只處理跨區域或阻斷性問題。成功匯入、儲存或計算可顯示短暫通知，但重要結果必須持續存在頁面中。

抽屜與 modal 保留：

- `role="dialog"` 與 `aria-modal="true"`。
- focus trap。
- Escape 關閉。
- 背景 inert。
- 關閉後焦點還原。

## 7. 響應式規則

### 大於 768px

- 日期區間雙欄。
- 月曆七欄。
- 薪資表單雙欄。
- 收入與扣款雙欄。
- 記錄使用右側抽屜。

### 768px 以下

- 流程摘要兩欄。
- 月曆轉為每週垂直清單。
- 薪資與請假表單單欄。
- 記錄抽屜占滿可用畫面。

### 480px 以下

- 摘要改為單欄或水平可滑動狀態列；實作選擇單欄，以降低隱藏資訊風險。
- 結果摘要與薪資明細單欄。
- 所有主要按鈕占滿容器寬度。
- 所有互動目標至少 44×44px。

## 8. 錯誤處理

- 空白、格式錯誤與超出範圍使用不同且具體的中文訊息。
- 錯誤訊息放在對應欄位附近。
- 成功、警告與錯誤同時使用圖示、標題與文字。
- 不以 modal 取代可在原位置修正的欄位錯誤。
- 計算錯誤或缺少必要資料時，不顯示看似有效的薪資結果。

## 9. 實作範圍

- `styles.css`：建立主要 Neo Brutal tokens、元件樣式與響應式規則。
- `index.html`：只調整必要結構、類別、狀態文字與來源註記。
- `motion.js`：收斂動畫，保留 `window.UiMotion` API。
- `service-worker.js`：只更新快取版本及實際新增的本機資產。
- `verify-ui-contract.mjs`：更新視覺契約，繼續保護 IDs、資料與可存取性。
- `uiverse.css`：若保留，只包含已選用且完成作用域隔離的樣式；禁止匯入整個 Galaxy。

本次不進行大規模 JavaScript 拆檔、框架遷移或資料模型重構。

## 10. 驗證

自動檢查：

- `node verify-payslip.mjs`
- `node verify-ui-contract.mjs`

視覺與互動檢查：

- 1440px 桌機。
- 768px 平板。
- 375px 手機。
- 日期產生與日期範圍錯誤。
- 2／4／8／清除快捷時數。
- 薪資計算與結果格式。
- 記錄新增、編輯、刪除、匯入與篩選。
- 鍵盤 Tab、Shift+Tab、Escape 與 focus 還原。
- modal、記錄抽屜與背景 inert。
- `prefers-reduced-motion`。
- PWA 離線載入與新版快取更新。

修改前後使用相同輸入比較薪資結果，結果必須完全一致。

## 11. 風險控制

- `index.html` 約 184 KB 且混合 UI 與商業邏輯；本次不並行進行大規模拆檔。
- 既有未提交修改必須逐檔保留，禁止重置或覆寫。
- Uiverse 原始動畫、通用類名與固定尺寸可能破壞現有介面；每個元件都必須重新命名、縮放、補強響應式與無障礙。
- Service Worker 可能顯示舊版資產；實作後必須更新快取版本並進行強制重新載入與離線驗證。

- 繁體中文在桌機、平板與手機皆清楚可見。
- 現有薪資計算、記錄、匯入、LocalStorage 與 PWA 行為維持相容。
- 自動檢查通過。
- 1440px、768px、375px 視覺與互動檢查通過。
